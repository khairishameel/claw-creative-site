Process

Three phases. No proposal decks.

Discovery, build, operate. Each priced separately, each fixed-fee. You decide whether to keep going at every gate.

Phase 01 · Days 1 to 14

Discovery

We sit with the human currently doing the work. We map inputs, decisions, exceptions, and what done looks like. We deliver a scoping doc: what's automatable, what's not, what it costs, what it takes. Fixed fee. Two weeks.

Phase 02 · Build

Build

We ship the system. Python, Playwright, LLM tools, your APIs. We build against an eval set drawn from real cases. Production-grade code, instrumented, alerted, documented. You own the repo.

Phase 03 · Optional

Operate

Monitoring, model upgrades, new capabilities. Monthly retainer, cancel anytime. Or take it in-house and run it yourself. Either is fine.

The pledge

What we will not do.

⛔

Strategy decks before code

If we can't write a scoping doc in two weeks, we shouldn't be in the room. No 90-day "AI strategy" engagements.

⛔

No-code-only builds

Real operations need real code. Zapier and Make are fine for trivial flows. Beyond that you need Python.

⛔

Lock-in middleware

You own the repo, the prompts, the eval set, the runbook. We don't bury logic in proprietary platforms.

⛔

Hype shipping

If your workflow doesn't need an LLM, we'll write the script without one. Tools fit problems, not the other way around.

⛔

Outsourcing

Every prompt, every line of Python, every eval, written in-house in Indianapolis.

⛔

Mystery pricing

Fixed fee per phase. You see the number before we start. No T&E surprise invoices.
